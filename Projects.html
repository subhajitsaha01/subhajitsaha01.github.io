Sahyog AI

Sahayog is an academic AI assistant for IITM BS students. It guides learners to lectures and resources without direct answers, promoting critical thinking and integrity. Integrated with the SEEK portal, it adapts to course content, offers web searches, and supports developers with conversation tracking—simplifying navigation and supporting self-paced learning. The primary features of this portal are :

  • As a Student, I want external resources to be recommended to me when no or insufficient results are found in the portal when asked a query, So that I can expand my learning and find the information I need.
  • As a Student, I want to be directed to the specific lecture number, references or documentation within.
    • the portal, So that I can quickly access relevant learning materials without searching extensively.
  • As a Developer, I want to maintain previous conversations, So that I can monitor and assess the agent’s responses effectively.
  • As a Student, I want the system to address FAQs and provide logistics and contact information, So that I can benefit from the collective knowledge if I need any kind of clarification and efficiently plan or get in touch without unnecessary delays.
Skills Used : Software Engineering Practices, Large Language Models (LLM), Google Gemini, Tavily, Flask

BDM Capstone Project

  • This project focuses on an online tuitions business run by a sole entity located at the residence. The business falls under the unorganized sector which is a B2C. Currently, the business serves students by providing online classes on a demand basis.
  • The business is was facing declining rate of prospective students who want online classes. Since the owner uses the UrbanPro platform for prospective lead generation, the owner was also looking for ways to increase the lead generation. At the same time the owner wanted to understand and track the operational expenditure and wanted to move towards efficient operational profit margin percentage to grow the business, so that it’s profitable in the long run. The owner also wanted to predict the annual income/revenue from this work.
  • Apart from that, owner also wanted to get hold of the operational expenses and what percentage is the actual revenue. Since, these issues identified has been persisting, it resulted in missed opportunities for income maximization and has hindered the business owner’s ability to make informed decisions, necessitating a structured approach to address these inefficiencies and capitalize on the business’s potential.
  • To combat the issues/challenges identified and to provide quality suggestion to Business owner on ways he could improve the business outcome, a thorough analysis of the tuition data was undertaken and a report was prepared containing the analysis. It will not only give insights on the current situation of the business and the revenue being generated, but also will help in understanding the best practices that should be followed. The issues will be addressed firstly by taking into proper documentation, all the data about inflow and outflow of revenue related to business related expenses. Also, by analysing the available data via different analytical approaches it was possible to obtain a fruitful outcome to solve the business challenges being faced by the owner.

Machine Learning Practise Project

  • The objective of this project was to implement the learnings learnt from the Machine Learning Practise Course and develop models capable of accurately predicting the crime categories based on this information.
  • The dataset offers a comprehensive snapshot of criminal activities within the city. It encompasses various aspects of each incident, including date, time, location, victim demographics and more.
  • The project was carried out through a Kaggle competition where the main outcome was to leverage machine learning techniques, with which the dataset was to be analyzed to predict crime categories and enhance law enforcement strategies and bolster public safety measures.
  • After proprocessing the dataset, I applied 7 models and did a comparative analysis of the training accuracy, testing accurcy and training times. I concluded that XGBoostClassifier was the best performing model in this dataset which reached around 95% testing accuracy on unknown dataset.

MusicStream | App Dev 2 Project

  • A multi user Music streaming application was made consisting of division of responsibilities among 3 users: Admin, Creator and User.
  • This web application has been made using by using Vue.js for frontend and Flask Microservices for backend seamlessly.
  • Admins maintain the access level of the users as well as maintaining the data for the albums, songs and playlists, while Creators will create/update and view the albums and song data.
  • Users can view the songs and albums data created by creators as well as create/update and view playlists of existing songs.
  • Security was ensured in the application using token-based authentication, enforced by Flask Security.
  • The application is made up of Flask REST Api which are restricted across various access levels of role-based authentication for strict data access, ensuring security at every call.
  • Additionally, Redis and Celery have been integrated with the application, to send regular reminders for Users to login as well as sending monthly reports to Creators for the content created by them.
  • Libraries Used for Implementation: HTML, CSS, JS, Vue js, Flask-Restful, Sqlite, Flask, Python, Redis, Celery, Chart js, Flask-Security

Grocery Store | App Dev 1 Project

  • The objective of this project is to build a grocery management system where there will be a user and a store manager.
  • The store manager can add various categories of products and can also add products under those categories.
  • The store manager has also the provisions of updating or deleting the data.
  • Meanwhile the user can buy products, view his/her cart, update the products in the cart as well buy the products.
  • Front end technologies: HTML, CSS, Bootstrap, JS - for working on front end views of the application and making it aesthetic and dynamic.
  • Backend technologies: Flask – This framework is used for designing the backend portion of the project mainly for server-side rendering.
  • Database Operations: Flask- SqlAlchemy – This library is used as an Object Relational Mapping framework for performing operations on the database table, mainly for performing Create, Read, Update and Delete operations.
  • REST Api: Flask – Restful – This library is used to build restful web services for performing the CRUD operations on database table, mainly for the Products and Purchase module.
  • Database: Sqlite – This embedded database is used as it is built in to Python and is highly efficient file-based database which doesn’t need a server for operations.
  • Programming Language: Python – All the backend modules have been written in python programming language.
  • Templating engine: Jinja2 – It is used as the templating engine for generating dynamic server pages along with Html content.
  • Analysis: Pandas and Matplotlib – It is used for generating the statistical analysis and graphs for admin summary page.
